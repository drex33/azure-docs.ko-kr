# <a name="configure-power-bi-visual-offer-listing-details"></a>시각적 제안 목록 세부 정보 Power BI 구성

이 페이지에서는 Microsoft AppSource에 나타나는 제품 이름, 설명, 링크, 연락처, 로고, 스크린샷 등의 제품 세부 정보를 정의할 수 있습니다. 이러한 세부 정보는 다양 한 언어로 제공 될 수 있습니다.

## <a name="languages"></a>언어

하나 이상의 지원 되는 언어에서 목록 세부 정보를 제공 합니다. 언어를 추가 하려면 **추가 언어 관리** 를 선택 합니다. 각 언어를 선택 하 여 목록 세부 정보를 추가 합니다.

:::image type="content" source="media/power-bi-visual/listing-languages.png" alt-text="제품 목록에 대 한 언어 선택 항목을 표시 합니다.":::

## <a name="marketplace-details"></a>마켓플레이스 세부 정보

- 여기에 입력 하는 **[이름은](/office/dev/store/reserve-solution-name)** 고객에 게 제품의 제목으로 표시 됩니다. 이 필드는 제품을 만들 때 입력 한 이름으로 미리 채워져 있지만 변경할 수 있습니다. 다른 언어에서와 같이 더 많은 이름을 예약 하려는 경우 [더 많은 이름 예약](power-bi-visual-manage-names.md)을 선택 합니다.
- 검색 결과 요약에 대 한 제품의 **요약** 을 입력 합니다. 이 설명은 Marketplace 검색 결과에서 사용할 수 있습니다.
- 제품에 대 한 철저 한 **설명을** 최대 3000 자까지 입력 합니다. 고객은 Marketplace 목록 개요에서 이 설명을 확인할 수 있습니다.
- 사용자가 제품을 찾을 수 있도록 최대 3 개의 **검색 키워드** 를 추가 합니다.

## <a name="marketplace-media"></a>마켓플레이스 미디어

제품에 사용할 아이콘과 이미지를 제공 합니다. 모든 이미지는 PNG 형식이어야 합니다. 이미지가 흐리면 제출이 거부됩니다.

>[!NOTE]
>파일을 업로드 하는 동안 문제가 발생 하는 경우 로컬 네트워크에서 https://upload.xboxlive.com 파트너 센터에서 사용 하는 서비스를 차단 하지 않는지 확인 합니다.

### <a name="logos"></a>로고

Microsoft AppSource 기본 페이지 및 검색 결과 페이지에 표시 되는 아이콘을 제공 합니다. PNG 형식 및 300 x 300 픽셀 이어야 합니다.

### <a name="screenshots"></a>스크린샷

하나 이상의 스크린샷 (최대 5 개)을 제공 합니다. 형식은 PNG 여야 하며 크기는 정확히 1366 px (너비) x 768 px (높이) 여야 합니다. 파일 크기는 1024 kb로 제한 됩니다. 더 많은 사용을 위해 각 스크린샷에서 텍스트 거품을 추가 하 여 표시 되는 주요 기능에 대 한 가치 제안을 분명히 하는 것이 좋습니다.

### <a name="additional-information-optional"></a>추가 정보 (선택 사항)

사용자 지정 시각적 개체에 대 한 사용자의 관심을 늘리려면 시각적 개체에 대 한 비디오에 **YouTube** 또는 **Vimeo** 링크를 제공 합니다. URL 형식은 https:// 또는 https:// 를 포함해야 합니다.

제품 목록을 최대한 효율적으로 만드는 방법에 대 한 팁은 [효과적인 목록을 만드는 팁]()를 참조 하세요.

리소스를 나열하는 추가 마켓플레이스는 [마켓플레이스 제품 목록에 대한 모범 사례](gtm-offer-listing-best-practices.md)를 참조하세요.

**초안 저장** 을 선택합니다.

추가 언어를 선택한 경우 페이지 맨 위에 있는 드롭다운 목록에서 각각을 선택 하 고 각각에 대해 위의 단계를 반복 합니다. 완료 되 면 왼쪽 탐색 메뉴의 **사용 가능** 에서 다음 탭으로 이동 합니다.
