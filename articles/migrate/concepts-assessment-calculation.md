# <a name="assessment-overview-migrate-to-azure-vms"></a>평가 개요(Azure VM으로 마이그레이션)

이 문서에서는 [Azure Migrate: 검색 및 평가](migrate-services-overview.md#azure-migrate-discovery-and-assessment-tool) 도구의 평가에 대한 개요를 제공합니다. 이 도구는 VMware 가상 및 Hyper-v 환경의 온-프레미스 서버 및 Azure로 마이그레이션하기 위한 물리적 서버를 평가할 수 있습니다.

## <a name="whats-an-assessment"></a>평가란 무엇인가요?

검색 및 평가 도구를 사용한 평가는 온-프레미스 서버를 Azure로 마이그레이션하기 위한 준비 상태를 측정하고 효과를 예측합니다.

> [!NOTE]
> Azure Government에서 [지원되는 대상](migrate-support-matrix.md#supported-geographies-azure-government) 평가 위치를 검토합니다. 평가의 VM 크기 권장 사항은 정부 클라우드 지역의 전용 VM 시리즈를 사용합니다. VM 형식에 대한 [자세한 정보](https://azure.microsoft.com/global-infrastructure/services/?regions=usgov-non-regional,us-dod-central,us-dod-east,usgov-arizona,usgov-iowa,usgov-texas,usgov-virginia&products=virtual-machines).

## <a name="types-of-assessments"></a>평가 형식

Azure Migrate: 검색 및 평가를 사용하여 만들 수 있는 평가에는 세 가지 형식이 있습니다.

***평가 형식** | **세부 정보**
--- | ---
**Azure VM** | 온-프레미스 서버를 Azure 가상 머신으로 마이그레이션하기 위한 평가. 이 평가 형식을 사용하여 Azure VM으로 마이그레이션하기 위한 [VMware](how-to-set-up-appliance-vmware.md) 및 [Hyper-V](how-to-set-up-appliance-hyper-v.md) 환경과 [물리적 서버](how-to-set-up-appliance-physical.md)에서 의 온-프레미스 서버를 평가할 수 있습니다.
**Azure SQL** | 온-프레미스 SQL Server를 VMware 환경에서 Azure SQL Database 또는 Azure SQL Managed Instance로 마이그레이션하기 위한 평가.
**Azure App Service** | VMware 환경에서 Azure App Service로 온-프레미스 웹앱을 마이그레이션하기 위한 평가.
**AVS(Azure VMware 솔루션)** | 온-프레미스 서버를 [AVS(Azure VMware 솔루션)](../azure-vmware/introduction.md)로 마이그레이션하기 위한 평가. 이 평가 유형을 사용하여 AVS(Azure VMware 솔루션)로 마이그레이션하기 위한 온-프레미스 [VMware VM](how-to-set-up-appliance-vmware.md)을 평가할 수 있습니다. [자세히 알아보기](concepts-azure-vmware-solution-assessment-calculation.md)

> [!NOTE]
> 검색 및 평가 도구에서 Azure VM의 수 또는 AVS 평가가 잘못된 경우, 총 평가 수를 클릭하여 모든 평가로 이동하고 Azure VM 또는 AVS 평가를 다시 계산합니다. 그러면 검색 및 평가 도구에 해당 평가 형식에 대한 올바른 개수가 표시됩니다. 

Azure Migrate를 사용하여 만든 평가는 데이터의 특정 시점 스냅샷입니다. Azure VM 평가는 다음과 같은 두 가지 크기 조정 기준 옵션을 제공합니다.

**평가 유형** | **세부 정보** | **Data**
--- | --- | ---
**성능 기반** | 수집된 성능 데이터를 기반으로 권장 사항을 만드는 평가 | VM 크기 권장 사항은 CPU 및 RAM 사용률 데이터를 기반으로 합니다.<br/><br/> 디스크 형식 권장 사항은 온-프레미스 디스크의 IOPS(초당 입출력 작업 수) 및 처리량을 기반으로 합니다. 디스크 형식은 Azure 표준 HDD, Azure 표준 SSD, Azure 프리미엄 디스크 및 Azure Ultra 디스크입니다.
**온-프레미스인 경우** | 권장하는 데 성능 데이터를 사용하지 않는 평가입니다. | VM 크기 권장 사항은 온-프레미스 온-프레미스 서버 크기를 기반으로 합니다.<br/><br> 권장 디스크 형식은 평가를 위해 스토리지 유형 설정에서 선택한 기반으로 합니다.

## <a name="how-do-i-run-an-assessment"></a>평가를 어떻게 실행할까요?

평가를 실행하는 방법에는 몇 가지가 있습니다.

- 경량 Azure Migrate 어플라이언스에서 수집한 서버 메타데이터를 사용하여 서버를 평가합니다. 어플라이언스는 온-프레미스 서버를 검색합니다. 그런 다음 서버 메타데이터 및 성능 데이터를 Azure Migrate에 보냅니다.
- 쉼표로 구분된 값(CSV) 형식으로 가져온 서버 메타데이터를 사용하여 서버를 평가합니다.

## <a name="how-do-i-assess-with-the-appliance"></a>어플라이언스를 어떻게 사용하여 평가할까요?

온-프레미스 서버를 발견하기 위해 Azure Migrate 어플라이언스를 배포하는 경우 다음 단계를 수행합니다.

1. Azure Migrate와 함께 사용할 Azure 및 온-프레미스 환경을 설정합니다.
1. 첫 평가를 위해 Azure 프로젝트를 만들고 검색 및 평가 도구를 추가합니다.
1. 경량 Azure Migrate 어플라이언스를 배포합니다. 이 어플라이언스는 지속해서 온-프레미스 서버를 검색하고 서버 메타데이터 및 성능 데이터를 Azure Migrate에 보냅니다. 어플라이언스를 VM 또는 물리적 서버로 배포합니다. 평가하려는 서버에는 아무것도 설치할 필요가 없습니다.

어플라이언스가 서버 검색을 시작한 후에는 평가하려는 서버를 그룹으로 수집하고 평가 형식이 **Azure VM** 인 그룹에 대한 평가를 실행할 수 있습니다.

[VMware](./tutorial-discover-vmware.md), [Hyper-v](./tutorial-discover-hyper-v.md)또는 [물리적 서버](./tutorial-discover-physical.md) 자습서에 따라 단계를 수행해 보세요.

## <a name="how-do-i-assess-with-imported-data"></a>어떻게 가져온 데이터로 평가할까요?

CSV 파일을 사용하여 서버를 평가하는 경우에는 어플라이언스가 필요하지 않습니다. 대신 다음 단계를 수행합니다.

1. Azure Migrate를 사용할 수 있도록 Azure 설정
1. 첫 평가를 위해 Azure 프로젝트를 만들고 검색 및 평가 도구를 추가합니다.
1. CSV 템플릿을 다운로드하고 서버 데이터를 추가합니다.
1. 템플릿을 Azure Migrate로 가져오기
1. 가져오기를 사용하여 추가된 서버를 발견하고, 그룹에 수집하고, 평가 형식이 **Azure VM** 인 그룹에 대한 평가를 실행합니다.

## <a name="what-data-does-the-appliance-collect"></a>어플라이언스는 어떤 데이터를 수집하나요?

평가에 Azure Migrate 어플라이언스를 사용하는 경우 [VMware](migrate-appliance.md#collected-data---vmware) 및 [Hyper-V](migrate-appliance.md#collected-data---hyper-v)를 위해 수집된 메타데이터 및 성능 데이터에 대해 알아봅니다.

## <a name="how-does-the-appliance-calculate-performance-data"></a>어플라이언스는 어떻게 성능 데이터를 계산하나요?

검색을 위해 어플라이언스를 사용하는 경우 다음 단계를 통해 계산 설정에 대한 성능 데이터를 수집합니다.

1. 어플라이언스는 실시간 샘플 포인트를 수집합니다.

    - **VMware VM**: 샘플 포인트는 20초마다 수집됩니다.
    - **Hyper-V VM**: 샘플 포인트는 30초마다 수집됩니다.
    - **물리적 서버**: 샘플 포인트는 5분마다 수집됩니다.

1. 어플라이언스는 샘플 포인트를 결합하여 VMware 서버 및 Hyper-v 서버에는 10분마다, 물리적 서버에는 5분마다 단일 데이터 요소를 만듭니다. 데이터 요소를 만들기 위해 어플라이언스는 모든 샘플에서 최곳값을 선택합니다. 그런 다음 데이터 요소를 Azure로 보냅니다.
1. 평가는 지난달의 모든 10분 데이터 요소를 저장합니다.
1. 평가를 만들 때 평가는 적절한 크기 조정에 사용할 적절한 데이터 요소를 식별합니다. 식별은 *성능 기록* 및 *백분위수 사용률* 에 대한 백분위수 값을 기준으로 합니다.

    - 예를 들어 성능 기록이 1주이고 백분위수 사용률이 95번째 백분위수인 경우, 평가는 지난주에 대한 10분 샘플 포인트를 정렬합니다. 오름차순으로 정렬하고 적절한 크기 조정의 95번째 백분위수 값을 선택합니다.
    - 95번째 백분위수 값은 99번째 백분위수 값을 선택할 때 포함될 수 있는 모든 이상값을 무시합니다.
    - 해당 기간의 최대 사용량을 선택하고 이상값을 누락하지 않으려면 백분위수 사용률로 99번째 백분위수를 선택해야 합니다.

1. 이 값은 어플라이언스가 수집하는 다음과 같은 메트릭을 위해 유효한 성능 사용률 데이터를 얻고자 쾌적 인자를 곱한 값입니다.

    - CPU 사용률
    - RAM 사용률
    - 디스크 IOPS(읽기 및 쓰기)
    - 디스크 처리량(읽기 및 쓰기)
    - 네트워크 처리량(in 및 out)

## <a name="how-are-azure-vm-assessments-calculated"></a>Azure VM 평가는 어떻게 계산되나요?

평가에서는 온-프레미스 서버의 메타데이터 및 성능 데이터를 사용하여 평가를 계산합니다. Azure Migrate 어플라이언스를 배포하는 경우 평가는 어플라이언스에서 수집하는 데이터를 사용합니다. 그러나 CSV 파일을 사용하여 가져온 평가를 실행하는 경우 계산을 위한 메타데이터를 제공합니다.

계산은 다음 세 단계에서 발생합니다.

1. **Azure 준비 상태 계산**: 서버가 Azure로 마이그레이션하는 데 적합한지를 평가합니다.
1. **크기 조정 계산 권장 사항 계산**: 컴퓨팅, 스토리지, 네트워크 크기를 예측합니다.
1. **월별 비용 계산**: 마이그레이션 후 Azure에서 서버를 실행하기 위한 예상 월별 컴퓨팅 비용 및 스토리지 비용을 계산합니다.

계산은 선행 순으로 진행합니다. 서버는 이전 단계를 통과하는 경우에만 이후 단계로 이동합니다. 예를 들어 서버가 Azure 준비 상태 단계에 실패하면 Azure에 적합하지 않은 것으로 표시됩니다. 해당 서버에 대한 크기 조정 및 비용 계산은 하지 않습니다.

## <a name="whats-in-an-azure-vm-assessment"></a>Azure VM 평가에는 무엇이 포함되어 있나요?

Azure VM 평가에 포함된 사항은 다음과 같습니다.

**속성** | **세부 정보**
--- | ---
**대상 위치** | 마이그레이션할 위치. 현재 평가는 다음 대상 Azure 지역을 지원합니다.<br/><br/> 오스트레일리아 동부, 오스트레일리아 남동부, 브라질 남부, 캐나다 중부, 캐나다 동부, 인도 중부, 미국 중부, 중국 동부, 중국 북부, 동아시아, 미국 동부, 미국 동부 2, 독일 중부, 독일 북동부, 일본 동부, 일본 서부, 한국 중부, 한국 남부, 미국 중북부, 북유럽, 미국 중남부, 동남 아시아, 인도 남부, 영국 남부, 영국 서부, US Gov 애리조나, US Gov 텍사스, US Gov 버지니아, 미국 중서부, 서유럽, 인도 서부, 미국 서부, 미국 서부 2.
**대상 스토리지 디스크(있는 그대로 크기 조정)** | Azure 스토리지에 사용할 디스크의 형식입니다. <br/><br/> 대상 스토리지 디스크를 프리미엄 관리형, 표준 SSD 관리형, 표준 HDD 관리형 또는 Ultra 디스크로 지정합니다.
**대상 스토리지 디스크(성능 기반 크기 조정)** | 대상 스토리지 디스크의 형식을 자동, 프리미엄 관리형, 표준 HDD 관리형, 표준 SSD 관리형 또는 Ultra 디스크로 지정합니다.<br/><br/> **자동**: 디스크 권장 사항은 디스크의 성능 데이터(IOPS 및 처리량)를 기반으로 합니다.<br/><br/>**프리미엄, 표준 또는 Ultra 디스크**: 평가에서는 선택한 스토리지 형식 내의 디스크 SKU를 권장합니다.<br/><br/> 99.9%의 단일 인스턴스 VM SLA(서비스 수준 약정)를 사용하려면 프리미엄 관리 디스크를 사용하는 것이 좋습니다. 그러면 모든 평가 내 디스크가 프리미엄 관리 디스크로 권장됩니다.<br/><br/> 높은 처리량, 높은 IOPS 및 일관되게 낮은 대기 시간 디스크 스토리지가 필요한 데이터 집약적 워크로드를 실행하려는 경우 Ultra 디스크 사용을 고려합니다.<br/><br/> 마이그레이션을 평가하는 경우 Azure Migrate는 관리 디스크만 지원합니다.
**Azure Reserved VM Instances** | 평가의 비용 예측이 고려할 수 있도록 [예약 인스턴스](https://azure.microsoft.com/pricing/reserved-vm-instances/)를 지정합니다.<br/><br/> '예약 인스턴스'를 선택하면 '할인율(%)' 및 'VM 작동 시간' 속성은 적용되지 않습니다.<br/><br/> 현재 Azure Migrate는 종량제 제안에 대해서만 Azure Reserved VM Instances를 지원합니다.
**크기 조정 기준** | Azure VM의 크기를 적절하게 조정하는 데 사용됩니다.<br/><br/> 있는 그대로 크기 조정 또는 성능 기반 크기 조정을 사용합니다.
**성능 기록** | 성능 기반 크기 조정을 사용하였습니다. 성능 기록은 성능 데이터를 평가할 때 사용되는 기간을 지정합니다.
**백분위 수 사용률** | 성능 기반 크기 조정을 사용하였습니다. 백분위 수 사용률은 적절한 크기 조정에 사용되는 성능 샘플의 백분위수 값을 지정합니다.
**VM 시리즈** | 적절한 크기 조정에 대해 고려할 Azure VM 시리즈입니다. 예를 들어 A 시리즈 VM이 필요한 프로덕션 환경이 Azure에 없는 경우 시리즈 목록에서 A 시리즈를 제외할 수 있습니다.
**쾌적 인자** | 평가 중에 사용되는 버퍼입니다. VM의 CPU, RAM, 디스크, 네트워크 데이터에 적용됩니다. 계절별 사용량, 짧은 성능 기록 및 향후 사용량 증가와 같은 문제를 고려합니다.<br/><br/> 예를 들어 사용률이 20%인 10코어 VM은 일반적으로 2코어 VM이라는 결과가 나옵니다. 2\.0의 쾌적 인자를 사용하면 4코어 VM이라는 결과가 대신 나옵니다.
**제품** | 등록한 [Azure 제품](https://azure.microsoft.com/support/legal/offer-details/)입니다. 평가는 해당 제품에 대한 비용을 추정합니다.
**통화** | 계정의 청구 통화입니다.
**할인(%)** | Azure 제품에 적용되는 구독별 할인입니다. 기본 설정은 0%입니다.
**VM 작동 시간** | 계속 실행되지 않는 Azure VM의 기간(월당 일수 및 일당 시간)입니다. 예상 비용은 기간을 기준으로 합니다.<br/><br/> 기본값은 월당 31일이며 일당 24시간입니다.
**Azure 하이브리드 혜택** | 소프트웨어 보증이 있고 [Azure 하이브리드 혜택](https://azure.microsoft.com/pricing/hybrid-use-benefit/)을 받을 수 있는지를 지정합니다. 설정의 기본값인 "예"인 경우 Windows 이외의 운영 체제에 대한 Azure 가격은 Windows VM을 기준으로 고려됩니다.
**EA 구독** | 비용 예측 시 EA(기업계약) 구독을 사용하도록 지정합니다. 구독에 적용되는 할인을 고려합니다. <br/><br/> 예약 인스턴스, 할인율(%), VM 작동 시간 속성을 기본 설정으로 놔둡니다.


Azure Migrate를 사용하여 평가를 만들기 위해 [모범 사례를 검토](best-practices-assessment.md)합니다.

## <a name="calculate-readiness"></a>준비 상태 계산

일부 서버는 Azure에서 실행하기에 적합하지 않습니다. Azure VM 평가는 모든 온-프레미스 서버를 평가하고 준비 범주를 할당합니다.

- **Azure 준비 완료** - 서버를 변경하지 않고 현재 상태대로 Azure로 마이그레이션할 수 있습니다. Azure를 완벽하게 지원하는 상태로 Azure에서 부팅됩니다.
- **조건부 Azure 준비 완료** - 서버 Azure에서 시작하지만, 전체 Azure 지원이 제공되지 않을 수도 있습니다. 예를 들어 Azure는 이전 버전의 Windows Server를 실행하는 서버를 지원하지 않습니다. 해당 서버를 Azure로 마이그레이션하기 전에 주의해야 합니다. 준비 문제를 해결하려면 평가에서 제안하는 수정 참고 자료를 따르세요.
- **Azure에 대해 준비되지 않음**: 서버가 Azure에서 시작되지 않습니다. 예를 들어 온-프레미스 서버의 디스크가 64TB를 초과하는 경우 Azure는 서버를 호스트할 수 없습니다. 마이그레이션 전에 수정 참고 자료를 따라 문제를 해결하세요.
- **준비 상태 알 수 없음**: Azure Migrate가 메타데이터 부족 때문에 서버 준비 상태를 확인할 수 없습니다.

평가 시 준비 상태를 계산하기 위해 다음 표에 요약된 서버 속성 및 운영 체제 설정을 검토합니다.

속성 | 세부 정보 | Azure 준비 상태
--- | --- | ---
**부팅 유형** | Azure는 부팅 형식이 BIOS인 VM을 지원하지만 UEFI인 VM은 지원하지 않습니다. | 부팅 형식이 UEFI이라면 조건부 준비 완료입니다.
**코어 수** | 각 서버의 코어 수는 Azure VM에서 지원하는 최대치인 128 이하여야 합니다.<br/><br/> 성능 기록을 사용할 수 있으면 Azure Migrate는 사용된 코어 수와 비교합니다. 평가 설정이 쾌적 인자를 지정하는 경우 사용된 코어 수와 쾌적 인자를 곱합니다.<br/><br/> 성능 기록이 없으면 Azure Migrate는 쾌적 인자를 적용하기 위해 할당된 코어 수를 사용합니다. | 코어 수가 한도를 초과하지 않으면 준비 완료

[자세히 알아봅니다](../virtual-machines/sizes.md).<br/><br/> 성능 기록을 사용할 수 있으면 Azure Migrate는 사용된 RAM 수와 비교합니다. 쾌적 인자가 지정된 경우 사용된 RAM 크기와 쾌적 인자를 곱합니다.<br/><br/> 기록이 없으면 할당된 RAM을 사용하여 쾌적 인자를 적용합니다.<br/><br/> | RAM 수가 한도를 초과하지 않으면 준비 완료
**스토리지 디스크** | 할당된 디스크 크기는 64TB 이하여야 합니다.<br/><br/> OS 디스크를 포함하여 서버에 연결된 디스크의 수는 65 미만이어야 합니다. | 디스크의 크기와 수가 한도를 초과하지 않으면 준비 완료

## <a name="calculating-sizing"></a>크기 조정 계산

서버가 Azure 준비 완료로 표시되면 평가는 Azure VM 평가에 크기 조정 권장 사항을 만듭니다. 해당 권장 사항은 Azure VM 및 디스크 SKU를 식별합니다. 크기 계산은 온-프레미스 크기 조정을 사용하는지 또는 성능 기반 크기 조정을 사용하는지에 따라 달라집니다.

### <a name="calculate-sizing-as-is-on-premises"></a>크기 조정 계산(온-프레미스에서 있는 그대로)

 온-프레미스에서 있는 그대로 크기 조정을 사용하는 경우 평가는 Azure VM 평가에서 VM 및 디스크의 성능 기록을 고려하지 않습니다.

- **컴퓨팅 크기 조정**: 평가는 온-프레미스에 할당된 크기에 따라 Azure VM SKU를 할당합니다.
- **스토리지 및 디스크 크기 조정**: 평가는 평가 속성에 지정된 스토리지 형식을 확인하고 적절한 디스크 형식을 권장합니다. 가능한 스토리지 형식은 표준 HDD, 표준 SSD, 프리미엄 및 Ultra 디스크입니다. 기본 스토리지 형식은 프리미엄입니다.
- **네트워크 크기 조정**: 평가는 온-프레미스 서버의 네트워크 어댑터를 고려합니다.

### <a name="calculate-sizing-performance-based"></a>크기 조정 계산(성능 기반)

Azure VM 평가에서 성능 기반 크기 조정을 사용하는 경우 평가는 다음과 같이 크기 조정 권장 사항을 만듭니다.

- 평가는 서버의 성능 기록을 고려하여 Azure의 VM 크기 및 디스크 형식을 식별합니다.
- CSV 파일을 사용하여 서버를 가져오는 경우 지정한 값이 사용됩니다. 이 방법은 특히 온-프레미스 서버를 초과 할당하고 사용률이 낮을 경우, 비용을 절감하기 위해 Azure VM의 크기를 적절히 조정하려는 경우에 특히 유용합니다.
- 성능 데이터를 사용하지 않으려면 이전 섹션에서 설명한 대로 크기 조정 기준을 온-프레미스에서 있는 그대로로 다시 설정합니다.

#### <a name="calculate-storage-sizing"></a>스토리지 크기 조정 계산

Azure VM 평가에서 스토리지 크기를 조정하는 경우 Azure Migrate는 서버에 연결된 각 디스크를 Azure 디스크에 매핑하려고 시도합니다. 크기 조정은 다음과 같이 작동합니다.

1. 평가는 디스크의 읽기 및 쓰기 IOPS를 추가하여 필요한 총 IOPS를 가져옵니다. 마찬가지로, 읽기 및 쓰기 처리량 값을 추가하여 각 디스크의 총처리량을 가져옵니다. 가져오기 기반 평가에는 개별 디스크 설정을 지정하지 않고 가져온 파일에서 디스크의 총 IOPS, 총처리량, 총 아니요를 제공하는 옵션이 있습니다. 가져온 파일의 디스크. 이렇게 하면 개별 디스크 크기 조정을 건너뛰고 제공된 데이터를 직접 사용하여 크기 조정을 계산하고 적절한 VM SKU를 선택합니다.

1. 스토리지 형식을 자동으로 지정한 경우 선택한 형식은 유효 IOPS 및 처리량 값을 기준으로 합니다. 평가는 Azure에서 디스크를 표준 HDD, 표준 SSD, 프리미엄 디스크 또는 Ultra 디스크에 매핑할지를 결정합니다. 스토리지 형식이 해당 디스크 형식 중 하나로 설정된 경우 평가는 선택한 스토리지 형식 내에서 디스크 SKU를 찾으려고 시도합니다.
1. 디스크는 다음과 같이 선택됩니다.
    - 평가에서 필요한 IOPS 및 처리량을 갖춘 디스크를 찾지 못하면 서버가 Azure에 적합하지 않은 것으로 표시됩니다.
    - 평가가 적합한 디스크 세트를 찾으면 평가 설정에 지정된 위치를 지원하는 디스크를 선택합니다.
    - 적합한 디스크가 여러 개인 경우 평가는 비용이 가장 낮은 디스크를 선택합니다.
    - 모든 디스크를 위한 성능 데이터를 사용할 수 없는 경우 구성 디스크 크기는 Azure에서 표준 SSD 디스크를 찾는 데 사용됩니다.

##### <a name="ultra-disk-sizing"></a>Ultra 디스크 크기 조정

Ultra 디스크의 경우 특정 디스크 크기에 대해 허용되는 IOPS 및 처리량 범위가 있으므로 크기 조정에 사용되는 논리는 표준 및 프리미엄 디스크와 다릅니다.
1. 세 가지 Ultra 디스크 크기가 계산됩니다. 
    - 디스크 크기 요구 사항을 충족할 수 있는 하나의 디스크(디스크 1)가 검색되었습니다.
    - 총 IOPS 요구 사항을 충족할 수 있는 하나의 디스크(디스크 2)가 검색되었습니다.
        - 프로비저닝할 IOPS = (원본 디스크 처리량) *1024/256
    - 총 처리량 요구 사항을 충족할 수 있는 하나의 디스크(디스크 3)가 검색되었습니다.
1. 3개의 디스크 중 최대 디스크 크기를 가진 하나가 검색되고 사용 가능한 다음의 [Ultra 디스크 제품](../virtual-machines/disks-types.md#disk-size)으로 반올림됩니다. 프로비저닝된 Ultra 디스크 크기입니다.
1. 프로비저닝된 IOPS는 다음 논리를 사용하여 계산됩니다.
    - 검색된 원본 처리량이 Ultra 디스크 크기에 허용되는 범위에 있는 경우 프로비저닝된 IOPS는 원본 디스크 IOPS와 동일합니다.
    - 그렇지 않은 경우 프로비저닝된 IOPS는 프로비저닝할 IOPS = (원본 디스크 처리량) *1024/256을 사용하여 계산됩니다.
1. 프로비저닝된 처리량 범위는 프로비저닝된 IOPS에 따라 다릅니다.


#### <a name="calculate-network-sizing"></a>네트워크 크기 조정 계산

Azure VM 평가의 경우 평가는 온-프레미스 서버에 연결된 네트워크 어댑터의 수 및 필요한 성능을 지원하는 Azure VM을 찾으려고 시도합니다.

- 온-프레미스 서버의 유효한 네트워크 성능을 얻기 위해 평가는 모든 네트워크 어댑터에서 서버(네트워크 출력)의 데이터 전송 속도를 집계합니다. 그런 다음 쾌적 인자를 적용합니다. 그 결괏값을 사용하여 필요한 네트워크 성능을 지원할 수 있는 Azure VM을 찾습니다.
- 네트워크 성능과 더불어, Azure VM이 필요한 네트워크 어댑터 수를 지원할 수 있는지도 고려합니다.
- 네트워크 성능 데이터를 사용할 수 없는 경우 평가는 VM 크기를 위한 네트워크 어댑터 수만 고려합니다.

#### <a name="calculate-compute-sizing"></a>컴퓨팅 크기 조정 계산

스토리지 및 네트워크 요구 사항이 계산된 후 평가는 CPU 및 RAM 요구 사항을 고려하여 Azure에 적합한 VM 크기를 찾습니다.

- Azure Migrate은 적절한 Azure VM 크기를 찾기 위해 유효한 사용률 코어와 RAM을 찾습니다.
- 적합한 크기가 없으면 해당 서버가 Azure에 적합하지 않은 것으로 표시됩니다.
- 적합한 크기가 있으면 Azure Migrate는 스토리지 및 네트워킹 계산을 적용합니다. 그런 다음 최종 VM 크기 권장 사항에 대해 위치 및 가격 책정 계층 설정을 적용합니다.
- 적합한 Azure VM 크기가 여러 개인 경우 비용이 가장 낮은 크기가 권장됩니다.

## <a name="confidence-ratings-performance-based"></a>신뢰 등급(성능 기반)

Azure Migrate에서 각각의 성능 기반 Azure VM 평가는 신뢰 등급과 연결됩니다. 등급의 범위는 1(최저)에서 5(가장 높음) 사이입니다. 신뢰 등급을 통해 Azure Migrate에서 제공하는 크기 권장 사항의 안정성을 예측할 수 있습니다.

- 신뢰 등급이 평가에 할당됩니다. 이 등급은 평가를 계산하는 데 필요한 데이터 요소의 가용성을 기반으로 합니다.
- 성능 기반 크기 조정의 경우 평가에는 다음이 필요합니다.
    - CPU 및 RAM을 위한 사용률 데이터.
    - 서버에 연결된 모든 디스크에 대한 디스크 IOPS 및 처리량 데이터.
    - 서버에 연결된 각 네트워크 어댑터에 대한 성능 기반 크기 조정을 처리하는 네트워크 I/O.

사용률을 사용할 수 없는 경우에는 크기 권장 사항이 불안정해질 수 있습니다.

> [!NOTE]
> 가져온 CSV 파일을 사용하여 평가된 서버에 대해서는 신뢰 등급이 지정되지 않습니다. 또한, 등급은 있는 그대로 온-프레미스 평가에 적용되지 않습니다.

### <a name="ratings"></a>등급

다음 표에서는 사용 가능한 데이터 요소의 백분율에 따른 평가 신뢰 등급을 보여 줍니다.

   **데이터 요소 가용성** | **신뢰 등급**
   --- | ---
   0-20% | 별 1개
   21-40% | 별 2개
   41-60% | 별 3개
   61-80% | 별 4개
   81-100% | 별 5개

### <a name="low-confidence-ratings"></a>낮은 신뢰 등급

여기에는 평가의 신뢰 등급이 낮아질 수 있는 이유가 나와 있습니다.

- 평가를 만드는 동안 환경을 프로파일링하지 않았습니다. 예를 들어 성능 기간을 1일로 설정하여 평가를 작성하는 경우 검색 시작 후 1일 이상 기다려야 모든 데이터 요소가 수집됩니다.
- 평가는 일부 또는 모든 서버에 대한 성능 데이터를 평가 기간에 수집할 수 없습니다. 높은 신뢰 등급의 경우 다음을 확인하세요. 
    - 평가 기간에 서버의 전원이 켜져 있는지 확인합니다.
    - 443 포트에서 아웃바운드 연결이 허용되는지 확인합니다.
    - Hyper-V 서버의 경우 동적 메모리가 사용하도록 설정되어 있는지 확인합니다. 
    
    신뢰 등급의 최신 변경 내용을 반영하려면 평가를 '다시 계산'하세요.

- 평가가 계산되는 기간에 일부 서버가 만들어졌습니다. 예를 들어 마지막 한 달의 성능 기록을 위한 평가를 만들려고 하는데, 일부 서버가 불과 일주일 전에 만들어졌습니다. 이 경우 새 서버의 성능 데이터는 전체 기간에 사용할 수 없으며 신뢰 등급은 낮습니다.

> [!NOTE]
> 평가의 신뢰 등급이 별 5개 미만인 경우 어플라이언스가 환경을 프로파일링하도록 하루 이상 기다린 다음, 평가를 다시 계산하는 것이 좋습니다. 그렇지 않으면 성능 기반 크기 조정이 불안정해질 수 있습니다. 이 경우 평가를 온-프레미스 크기 조정으로 전환하는 것이 좋습니다.

## <a name="calculate-monthly-costs"></a>월별 비용 계산

크기 조정 권장 사항이 완료된 후 Azure Migrate의 Azure VM 평가는 마이그레이션 후의 컴퓨팅 비용 및 스토리지 비용을 계산합니다.

### <a name="compute-cost"></a>컴퓨팅 비용
Azure Migrate는 권장되는 Azure VM 및 Azure 청구 API를 사용하여 서버 월별 비용을 계산합니다.

계산할 때는 다음을 고려합니다.
- 운영 체제
- Software Assurance
- 예약 인스턴스
- VM 작동 시간
- Location
- 통화 설정

평가는 모든 서버에서 비용을 집계하여 월별 총 컴퓨팅 비용을 계산합니다.

### <a name="storage-cost"></a>스토리지 비용
서버에 연결된 모든 디스크의 월별 비용을 집계하여 해당 서버의 월별 스토리지 비용을 계산합니다.

#### <a name="standard-and-premium-disk"></a>표준 및 프리미엄 디스크
표준 또는 프리미엄 디스크 비용은 선택한/권장되는 디스크 크기를 기준으로 계산됩니다. 

#### <a name="ultra-disk"></a>Ultra disk 

Ultra 디스크 비용은 프로비저닝된 크기, 프로비저닝된 IOPS 및 프로비저닝된 처리량을 기준으로 계산됩니다. [자세한 정보](https://azure.microsoft.com/pricing/details/managed-disks/)

비용은 다음 논리를 사용하여 계산됩니다. 
- 디스크 크기 비용은 프로비저닝된 디스크 크기에 디스크 용량의 시간당 가격을 곱하여 계산됩니다.
- 프로비저닝된 IOPS 비용은 프로비저닝된 IOPS에 시간당 프로비저닝된 IOPS 가격을 곱하여 계산됩니다.
- 프로비저닝된 처리량 비용은 프로비저닝된 처리량에 시간당 프로비저닝된 처리량 가격을 곱하여 계산됩니다.
- Ultra 디스크 VM 예약 요금은 총 비용에 합산되지 않습니다. [자세한 내용](https://azure.microsoft.com/pricing/details/managed-disks/)

평가는 모든 서버의 스토리지 비용을 집계하여 월별 총 스토리지 비용을 계산합니다. 현재 계산에서는 평가 설정에 지정된 제안을 고려하지 않습니다.

비용은 평가 설정에 지정된 통화로 표시됩니다.
