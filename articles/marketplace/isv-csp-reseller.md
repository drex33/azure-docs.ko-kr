# <a name="isv-to-cloud-reseller-private-offers"></a>ISV에서 클라우드로 Reseller 프라이빗 제안

> [!IMPORTANT]
> 이 기능은 현재 공개 미리 보기로 제공됩니다.

## <a name="overview"></a>개요

프라이빗 제안을 통해 ISV(Independent Software Vendor) 및 CSP 파트너(Cloud Resellers)는 각 엔터티의 비즈니스 요구 사항에 맞는 시간 제한 사용자 지정 여백을 만들어 수익을 늘릴 수 있습니다.

ISV로, CSP 파트너에 대한 판매 가격을 만드는 데 사용할 여유 및 기간을 지정할 수 있습니다. 파트너가 고객에게 판매할 때 Microsoft는 고객에게 판매 대금을 지급합니다.

CSP 파트너는 파트너 센터 UI 또는 API를 통해 사용 가능한 모든 여백을 검색할 수 있습니다. 판매한 모든 판매의 경우 판매 금액으로 Microsoft로부터 청구서를 받게 됩니다. 계속해서 고객 가격을 설정하고 마켓플레이스 외부에서 고객에게 청구서를 청구합니다. [ISV로 구성된 여백 검색에서](/partner-center/csp-commercial-marketplace-margins)프라이빗 제안의 CSP 파트너 경험에 대해 자세히 알아보세요.

:::image type="content" source="media/isv-csp-cloud/isv-private-offer-experience.png" alt-text="ISV 프라이빗 제안 환경의 진행 상황을 표시합니다.":::

:::image type="content" source="media/isv-csp-cloud/csp-private-offer-experience.png" alt-text="CSP 프라이빗 제안 환경의 진행 상황을 표시합니다.":::

> [!NOTE]
> Azure Marketplace 통해 사용할 수 없도록 Microsoft AppSource 제안을 게시하고 CSP(클라우드 솔루션 공급자) 프로그램의 파트너가 판매하도록 선택한 경우 CSP 파트너가 포털에서 구매할 수 있도록 제안이 표시될 때 지연이 발생할 수 있습니다. CSP 파트너가 거래할 수 있는 제안이 없다고 생각되면 [고객 지원](./support.md) 담당자에게 문의하세요.

## <a name="prerequisites-to-create-a-private-offer-for-cloud-resellers"></a>Cloud Resellers에 대한 프라이빗 제안을 만들기 위한 필수 조건

클라우드 대리점에 대한 프라이빗 제안을 만들려면 다음 필수 구성을 충족해야 합니다.

1. 파트너 센터 상업용 Marketplace 계정을 만들었습니다.
2. 계정이 상업용 Marketplace 프로그램에 등록되어 있습니다.
3. Azure Marketplace 거래 가능하고 공개적으로 사용할 수 있는 제안을 게시했습니다.
4. 제안이 Cloud Reseller 채널에 옵트인됩니다.
5. Microsoft의 CSP(클라우드 솔루션 공급자) 프로그램에 속한 Cloud Reseller를 위한 프라이빗 제안을 만들고 있습니다.

### <a name="supported-offer-types"></a>지원되는 제안 유형

거래 가능한 모든 마켓플레이스 제안 유형에 대해 프라이빗 제안을 만들 수 있습니다. 여기에는 SaaS, Azure Virtual Machines 및 Azure 애플리케이션이 포함됩니다.

> [!NOTE]
> 여백은 제안이 사용할 수 있는 모든 사용자 지정 미터 크기에 적용됩니다. 여백은 연결된 Azure 인프라 하드웨어 요금이 아니라 설정한 소프트웨어 요금에만 적용됩니다.

## <a name="private-offers-dashboard"></a>프라이빗 제안 대시보드

파트너 센터 왼쪽 탐색 메뉴의 **프라이빗 제안** 대시보드는 프라이빗 제안을 만들고 관리하는 중앙 집중식 위치입니다. 이 대시보드에는 다음 두 개의 탭이 있습니다.

1. **고객:** 기능이 2021년 Fall에 프라이빗 미리 보기로 전환되면 고객을 위한 프라이빗 제안에 대해 자세히 알아봅니다.
2. **Cloud Resellers:** 다음을 수행할 수 있는 Cloud Reseller 프라이빗 제안 대시보드를 엽니다.

    - 새 프라이빗 제안 만들기
    - 모든 프라이빗 제안의 상태 보기
    - 기존 프라이빗 제안 복제
    - 프라이빗 제안 철회
    - 프라이빗 제안 삭제

## <a name="create-a-new-private-offer-for-cloud-resellers"></a>클라우드 대리점에 대한 새 프라이빗 제안 만들기

1. 파트너 센터에 로그인합니다.
2. 왼쪽 탐색 메뉴에서 **프라이빗 제안을** 선택하여 대시보드를 엽니다.
3. Cloud **Resellers 탭을** 선택합니다.
4. **+ 새 프라이빗 제안을** 선택합니다.
5. 프라이빗 제안 이름을 입력합니다. 파트너 센터 내에서 프라이빗 제안을 참조하는 데 사용할 설명이 포함된 이름입니다. 이 이름은 Cloud Resellers에 표시되지 않습니다.  

### <a name="offer-setup"></a>제안 설정

제품 설정 페이지에서는 프라이빗 제품 약관, 알림 연락처, 가격 책정 및 Cloud Resellers를 정의할 수 있습니다.

1. **프라이빗 제품 약관에** 따라 클라우드 재판매인에서 프라이빗 제안을 검색하고 판매할 수 있는 기간이 결정됩니다.

    - 프라이빗 제안을 즉시 시작하려면 **시작 날짜를** "가능한 한 빨리"로 선택합니다. 모든 필수 조건이 충족되면 제출한 후 15분 이내에 프라이빗 제안을 사용할 수 있습니다. 프라이빗 제품이 종량제 제품의 기존 고객으로 확장되면 프라이빗 가격이 한 달 전체에 적용됩니다.
    - 프라이빗 제안을 다음 달에 시작하려면 특정 **월을** 선택하고 선택합니다. 이 옵션의 시작 날짜는 항상 월의 첫 번째 날짜입니다.
    - 프라이빗 제안의 종료 날짜 에 대한 **월을** 선택합니다. 이 날짜는 항상 월의 마지막 날짜입니다.

2. 프라이빗 제안의 상태에 대한 메일 업데이트를 받으려면 **알림 연락처로** 최대 5개의 이메일을 제공합니다. 이러한 이메일은 프라이빗 제안이 **라이브**, **종료** 또는 **철회로** 이동하면 전송됩니다.

3. 프라이빗 제품에서 최대 10개 제품/플랜에 대한 **가격 책정** 여백 비율을 구성합니다. Cloud Reseller가 받는 수익은 마켓플레이스에서 플랜의 정가에서 백분율로 떨어지게 됩니다.

    - **+ 제안/계획 추가를** 선택하여 프라이빗 제안을 제공하려는 제안/플랜을 선택합니다.
    - 제안 수준(해당 제안 아래의 모든 현재 및 향후 플랜에는 프라이빗 가격이 연결됩니다.) 또는 계획 수준에서 프라이빗 제안을 제공하도록 선택합니다(선택한 플랜에만 프라이빗 가격이 연결됩니다).
    - 최대 10개 제안/플랜을 선택한 다음, **를 추가합니다.**
    - Cloud Resellers에 제공할 여백 비율을 입력합니다. 제공하는 수익은 마켓플레이스에서 플랜의 정가에서 백분율로 계산됩니다.

    > [!NOTE]
    > Microsoft AppSource 또는 Azure Marketplace 거래할 수 있는 제안/계획만 선택 메뉴에 표시됩니다.

4. 프라이빗 제안을 판매할 권한이 있는 **Cloud Resellers를** 선택합니다.

    1. **+클라우드 대리점 추가를** 선택합니다.
    2. 이름/테넌트 ID로 Cloud Reseller를 검색합니다. 또는 지역, 기술 또는 역량과 같은 필터를 적용하여 검색합니다.
    3. Cloud Resellers를 선택하고 **추가를** 선택합니다.

    > [!NOTE]
    > - Microsoft의 CSP(클라우드 솔루션 공급자) 프로그램에 속한 Cloud Resellers만 선택할 수 있습니다. 
    > - 프라이빗 제안이 종료되면 권한을 부여하는 Cloud Resellers가 마켓플레이스 제안을 정가로 계속 판매할 수 있습니다.
    > - 프라이빗 제안은 최대 400개의 Cloud Resellers 테넌트까지 확장할 수 있습니다.

### <a name="review-and-submit"></a>검토 및 제출

이 페이지에서는 제공한 모든 정보를 검토할 수 있습니다.

제출된 후에는 프라이빗 제안을 수정할 수 없습니다. 정보가 정확한지 확인합니다.

준비가 되면 **제출을** 클릭합니다. 프라이빗 제안의 상태를 볼 수 있는 대시보드로 돌아갑니다. 프라이빗 제안이 라이브되면 알림 연락처가 전자 메일을 받게 됩니다.

## <a name="view-private-offers-status"></a>프라이빗 제안 상태 보기

프라이빗 제안의 상태를 보려면 다음을 수행합니다.

1. 왼쪽 탐색 메뉴에서 **프라이빗 제안을** 선택하여 대시보드를 엽니다.
2. Cloud **Resellers 탭을** 선택합니다.
3. **상태** 열을 확인합니다.

프라이빗 제안의 상태는 다음 중 하나입니다.

- **초안:** 프라이빗 제안을 만드는 프로세스를 시작했지만 아직 제출하지 않았습니다.
- **진행 중:** 프라이빗 제안을 제출했으며 현재 시스템에 게시되고 있습니다.
- **라이브:** 프라이빗 제안은 Cloud Resellers에서 검색 가능하고 거래할 수 있습니다.
- **종료:** 프라이빗 제안이 만료되거나 종료 날짜가 지났습니다.

## <a name="clone-a-private-offer"></a>프라이빗 제안 복제

프라이빗 제안을 복제하면 새 프라이빗 제안을 빠르게 만들 수 있습니다.

1. 왼쪽 탐색 메뉴에서 **프라이빗 제안을** 선택하여 대시보드를 엽니다.
2. Cloud **Resellers 탭을** 선택합니다.
3. 복제하려는 프라이빗 제안의 확인란을 선택합니다.
4. **복제** 를 선택합니다.
5. 새 프라이빗 제안 이름을 입력합니다.
6. **복제** 를 선택합니다.
7. 필요에 따라 제안 설정 페이지에서 세부 정보를 편집합니다.
8. 새 개인 제품을 **제출** 합니다.

## <a name="withdraw-a-private-offer"></a>개인 제품 철회

개인 제품을 입출금 계좌 클라우드 대리점은 더 이상 더 이상 여백을 받고 모든 향후 구매는 정가에 있습니다.

> [!IMPORTANT]
> 사설 제품은 클라우드 재판매인이 고객에 게 판매 하지 않은 경우에만 제거할 수 있습니다.

비공개 제품을 철회 하려면:

1. 왼쪽 탐색 메뉴에서 **비공개 제안** 을 선택 하 여 대시보드를 엽니다.
2. **클라우드 대리점** 탭을 선택 합니다.
3. 철회 하려는 개인 제품의 확인란을 선택 합니다.
4. **철회** 를 선택 합니다.
5. **요청 철회** 를 선택 합니다.
6. 개인 제안이 성공적으로 인출 되 면 알림 담당자가 전자 메일을 받게 됩니다.

## <a name="delete-a-private-offer"></a>비공개 제품 삭제

초안 상태에서 비공개 제품을 삭제 하려면 다음을 수행 합니다.

1. 왼쪽 탐색 메뉴에서 **비공개 제안** 을 선택 하 여 대시보드를 엽니다.
2. **클라우드 대리점** 탭을 선택 합니다.
3. 삭제 하려는 개인 제품의 확인란을 선택 합니다.
4. **삭제** 를 선택합니다.
1. **확인** 을 선택합니다.

## <a name="find-additional-details"></a>추가 세부 정보 찾기

개인 제공 게시가 진행 중인 동안에는 현재 상태에 대 한 추가 세부 정보를 볼 수 있습니다.

1. 왼쪽 탐색 메뉴에서 **비공개 제안** 을 선택 하 여 대시보드를 엽니다.
2. **클라우드 대리점** 탭을 선택 합니다.
3. **상태** 열에서 개인 제품의 "진행 중" 하이퍼링크를 선택 합니다.

추가 세부 정보는 다음 중 하나가 됩니다.

- **클라우드 재판매인 권한 부여 진행** 중: 현재 제공 된 클라우드 대리점에 제품을 판매할 수 있는 권한을 부여 하 고 있습니다.
- **비공개 제안 게시 진행** 중: 현재 지정 된 클라우드 대리점의 비공개 가격을 게시 하는 중입니다.
- **라이브**: 개인 제품은 이제이 클라우드 대리점에 대해 살고 있습니다.

## <a name="reporting-on-private-offers"></a>개인 제품에 대 한 보고

Microsoft에서 요금을 지불 하는 지급 양과 에이전시 요금은 연결 된 여백이 있는 품목에 대해 여백이 적용 된 후의 가격을 기준으로 합니다.
