# <a name="confidence-score"></a>신뢰도 점수

사용자 쿼리가 프로젝트(기술 자료라고도 함)와 일치하는 경우 질문 답변은 신뢰도 점수와 함께 관련 답변을 반환합니다. 이 점수는 응답이 지정된 사용자 쿼리에 올바르게 일치한다는 신뢰도를 나타냅니다.

신뢰도 점수는 0과 100 사이의 숫자입니다. 점수가 100이면 정확히 일치할 가능성이 높지만 점수가 0이면 일치하는 응답을 찾지 못한 것입니다. 이 점수가 높을수록 응답의 신뢰도가 높습니다. 지정된 쿼리에 대해 여러 응답이 반환될 수 있습니다. 이 경우 응답은 높은 신뢰도 점수부터 반환됩니다.

다음 표에서는 지정된 점수와 연결된 일반적인 신뢰도를 보여 줍니다.

|점수 값|점수 의미|예제 쿼리|
|--|--|--|
|90 - 100|사용자 쿼리와 KB 질문이 거의 정확히 일치합니다.|
|> 70|높은 신뢰도 - 일반적으로 사용자의 쿼리에 완벽하게 좋은 응답|
|50 - 70|중간 신뢰도 - 일반적으로 사용자 쿼리의 기본 의도에 적절하게 맞는 응답|
|30 - 50|낮은 신뢰도 - 일반적으로 사용자의 의도에 부분적으로 부합되는 관련된 응답|
|< 30|매우 낮은 신뢰도 - 일반적으로 사용자 쿼리에 대한 응답이 아니며 일치하는 일부 단어나 구 포함 |
|0|일치하는 부분이 없으므로 응답은 반환되지 않습니다.|

## <a name="choose-a-score-threshold"></a>점수 임계값 선택

위의 표에서는 질문 대답을 통해 쿼리할 때 발생할 수 있는 점수의 범위를 보여줍니다. 그러나 모든 프로젝트가 다르고 다양한 유형의 단어, 의도 및 목표가 있으므로 가장 적합한 임계값을 테스트하고 선택하는 것이 좋습니다. 기본적으로 임계값은 로 `0` 설정되므로 가능한 모든 답변이 반환됩니다. 대부분의 프로젝트에서 작동해야 하는 권장 임계값은 **50입니다.**

임계값을 선택할 때 **정확도와** **적용 범위** 간의 균형을 염두에 두고 요구 사항에 따라 임계값을 조정합니다.

- 사용자 시나리오에서 **정확도**(또는 정밀도)가 더 중요한 경우 임계값을 늘립니다. 이러한 방식으로 진행하면 응답을 반환할 때마다 훨씬 더 신뢰할 수 있는 사례가 되고, 사용자가 원하는 응답이 될 가능성이 높아집니다. 이 경우 더 많은 질문에 응답이 제공되지 않을 수 있습니다. 

- **적용 범위**(또는 재현율)가 좀 더 중요하고 가능한 한 많은 질문에 응답하려는 경우(사용자 질문에 부분적으로만 연관되어 있더라도) 임계값을 낮춥니다. 응답이 사용자의 실제 쿼리에 대한 응답이 아니지만 관련된 다른 응답을 제공하는 경우가 더 많을 수 있습니다.

## <a name="set-threshold"></a>임계값 설정

임계값 점수를 [REST API JSON 본문의](../quickstart/sdk.md)속성으로 설정합니다. 즉, REST API 대한 각 호출에 대해 설정합니다.

## <a name="improve-confidence-scores"></a>신뢰도 점수 개선

사용자 쿼리에 대한 특정 응답의 신뢰도 점수를 향상하기 위해 사용자 쿼리를 해당 응답의 대체 질문으로 기술 자료에 추가할 수 있습니다. 대/소문자를 구분하지 않으면 [동의어를](../tutorials/adding-synonyms.md) 사용하여 프로젝트의 키워드에 동의어를 추가할 수도 있습니다.

## <a name="similar-confidence-scores"></a>유사한 신뢰도 점수

여러 응답의 신뢰도 점수가 유사한 경우 쿼리가 너무 일반적이어서 여러 답변과 동일한 가능성으로 일치할 수 있습니다. 모든 QnA 엔터티에 고유한 의도가 있도록 QnA를 더 효과적으로 구성합니다.

## <a name="confidence-score-differences-between-test-and-production"></a>테스트와 프로덕션 간의 신뢰도 점수 차이

응답의 신뢰도 점수는 콘텐츠가 동일한 경우에도 프로젝트의 테스트 버전과 배포된 버전 간에 무시할 수 있습니다. 테스트와 배포된 프로젝트의 콘텐츠가 서로 다른 Azure Cognitive Search 인덱스 안에 있기 때문입니다.

테스트 인덱스는 프로젝트의 모든 질문 및 답변 쌍을 보유합니다. 테스트 인덱스를 쿼리할 때 쿼리가 전체 인덱스에 적용된 다음 결과가 해당 특정 프로젝트의 파티션으로 제한됩니다. 테스트 쿼리 결과가 프로젝트의 유효성을 검사하는 기능에 부정적인 영향을 미치는 경우 다음을 수행할 수 있습니다.
* 다음 중 하나를 사용하여 프로젝트를 구성합니다.
    * 하나의 리소스가 하나의 프로젝트로 제한됨: 단일 언어 리소스(및 결과 Azure Cognitive Search 테스트 인덱스)를 프로젝트/기술 자료로 제한합니다.
    * 테스트용 리소스 1개, 프로덕션용 리소스 1개, 테스트용 리소스 1개, 테스트용 리소스 1개(자체 테스트 및 프로덕션 인덱스 사용) 및 프로덕션용 리소스(자체 테스트 및 프로덕션 인덱스 사용) 2개
* 테스트 프로젝트와 프로덕션 프로젝트를 모두 쿼리할 때는 항상 동일한 매개 변수를 사용합니다.

프로젝트를 배포하면 프로젝트의 질문 및 답변 콘텐츠가 테스트 인덱스에서 Azure Search의 프로덕션 인덱스로 이동합니다.

다른 지역에 프로젝트가 있는 경우 각 지역은 자체 Azure Cognitive Search 인덱스를 사용합니다. 다른 인덱스가 사용되므로 점수가 정확하게 일치하지 않습니다.

## <a name="no-match-found"></a>일치하는 항목 없음

순위 지정자가 일치하는 것을 찾을 수 없는 경우 신뢰도 점수 0.0 또는 "None"이 반환되고 기본 응답이 반환됩니다. [기본 응답](../how-to/change-default-answer.md)을 변경할 수 있습니다.
