# <a name="defender-for-iot-glossary-for-organizations"></a>조직용 Defender for IoT 용어집

이 용어집에서는 Azure Defender for IoT 플랫폼에 대한 중요 용어 및 개념에 대해 간략하게 설명합니다. **자세한 정보** 링크를 선택하여 용어집의 관련 용어로 이동합니다. 이렇게 하면 제품 도구를 보다 신속하게 학습하고 사용할 수 있습니다.

<a name="glossary-a"></a>

## <a name="a"></a>A

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **액세스 그룹** | 액세스 그룹 규칙을 만들어 대기업의 사용자 액세스 요구 사항을 지원합니다.<br /><br />규칙을 사용하면 관련 사업부, 지역, 사이트 및 영역의 특정 사용자 역할에 대해 Defender for IoT 온-프레미스 관리 콘솔의 보기 및 구성 액세스 권한을 제어할 수 있습니다.<br /><br />예를 들어, Active Directory 그룹의 보안 분석가가 서유럽 자동차 데이터에는 액세스할 수 있지만 아프리카의 데이터에는 액세스하지 못하게 할 수 있습니다. | **[온-프레미스 관리 콘솔](#o)** <br /><br />**[사업부](#b)** |
| **액세스 토큰** | 액세스 토큰을 생성하여 Defender for IoT REST API에 액세스합니다. | **[API](#glossary-a)** |
| **경고 이벤트 승인** | 검색된 이벤트에 대해 경고를 한 번만 숨기도록 Defender for IoT에 지시합니다. 이벤트가 다시 검색되면 경고가 다시 트리거됩니다. | **[경고](#glossary-a)<br /><br />[경고 이벤트 알아보기](#l)<br /><br />[경고 이벤트 음소거](#m)** |
| **경고** | 승인된 네트워크 동작과의 차이, 네트워크 변칙 또는 의심스러운 네트워크 활동과 트래픽과 관련하여 Defender for IoT가 트리거하는 메시지입니다. | **[전달 규칙](#f)<br /><br />[제외 규칙](#e)<br /><br />[시스템 알림](#s)** |
| **경고 설명** | 보안 분석가 및 관리자가 경고 메시지에 추가하는 설명입니다. 예를 들어, 경고 설명은 수행할 완화 작업 또는 이벤트와 관련하여 연락할 담당자의 이름에 대한 지침을 제공할 수 있습니다.<br /><br />경고를 검토하는 사용자는 이벤트 상태를 가장 잘 반영하는 설명을 선택하거나 경고를 조사하기 위해 수행할 단계를 선택할 수 있습니다. | **[경고](#glossary-a)** |
| **변칙 엔진** | 비정상적인 M2M(머신 간) 통신 및 동작을 검색하는 Defender for IoT 엔진입니다. 예를 들어, 엔진은 과도한 SMB 로그인 시도를 검색할 수 있습니다. 변칙 경고는 이러한 이벤트가 검색될 때 트리거됩니다. | **[Defender for IoT 엔진](#d)** |
| **API** | 외부 시스템은 Defender for IoT에서 검색된 데이터에 액세스하고, SSL 연결을 통해 외부 REST API를 사용하여 작업을 수행할 수 있습니다. | **[액세스 토큰](#glossary-a)** |
| **공격 벡터 보고** | 악용 가능 엔드포인트의 취약성 체인에 대한 실시간 그래픽 표현입니다.<br /><br />보고서를 사용하면 공격 순서에서 완화 작업의 효과를 평가하여 확인할 수 있습니다. 예를 들어, 시스템 업그레이드가 공격 체인을 차단하여 공격자의 경로를 중단하는지 여부 또는 대체 공격 경로가 유지되는지 여부를 평가할 수 있습니다. 이를 통해 재구성 및 완화 작업의 우선 순위가 결정됩니다. | **[위험 평가 보고서](#r)** |

## <a name="b"></a>b

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **사업부** | 특정 산업에 따른 비즈니스의 논리적 구성입니다.<br /><br />예를 들어, 유리 공장과 플라스틱 공장을 포함하는 글로벌 기업은 서로 다른 두 사업부로 관리할 수 있습니다. 특정 사업부에 대한 Defender for IoT 사용자의 액세스를 제어할 수 있습니다. | **[온-프레미스 관리 콘솔](#o)<br /><br />[액세스 그룹](#glossary-a)<br /><br />[사이트](#s)<br /><br />[영역](#z)** |
| **기초** | 승인된 네트워크 트래픽, 프로토콜, 명령 및 디바이스. Defender for IoT는 네트워크 기준에서의 편차를 식별합니다. 데이터 마이닝 보고서를 생성하여 승인된 기준 트래픽을 확인합니다. | **[데이터 마이닝](#d)<br /><br />[학습 모드](#l)** |

## <a name="c"></a>C

| 용어 | Description | 자세한 정보 |

## <a name="d"></a>D

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **데이터 마이닝** | 네트워크 디바이스에 대한 포괄적이고 세부적인 보고서를 생성합니다.<br /><br />- **SOC 인시던트 응답**: 즉각적인 인시던트 대응을 처리하는 데 유용한 실시간으로 보고서입니다. 예를 들어, 보고서에 패치가 필요할 수 있는 디바이스를 나열할 수 있습니다.<br /><br />- **포렌식**: 조사 보고서에 기록된 데이터를 기준으로 하는 보고서입니다.<br /><br />- **IT 네트워크 무결성**: 전반적인 네트워크 보안을 개선하는 데 도움이 되는 보고서입니다. 예를 들어, 보고서에 약한 인증 자격 증명이 있는 디바이스를 나열할 수 있습니다.<br /><br />- **표시 유형**: 네트워크의 모든 기본 매개 변수를 확인하기 위한 모든 쿼리 항목을 포함하는 보고서입니다.<br /><br />읽기 전용 사용자가 볼 수 있는 데이터 마이닝 보고서를 저장합니다. | **[기준](#b)<br /><br />[보고서](#r)** |
| **Defender for IoT 플랫폼** | Defender for IoT 센서 및 온-프레미스 관리 콘솔에 설치되는 Defender for IoT 솔루션입니다. | **[센서](#s)<br /><br />[온-프레미스 관리 콘솔](#o)** |
| **디바이스 맵** | Defender for IoT에서 감지한 네트워크 디바이스를 그래픽으로 표현한 것입니다. 디바이스 간 연결 및 각 디바이스에 대한 정보를 표시합니다. 맵을 사용하여 다음을 수행합니다.<br /><br />- 중요한 디바이스 정보를 검색하고 제어합니다.<br /><br />- 네트워크 조각을 분석합니다.<br /><br />- 디바이스 세부 정보 및 요약을 내보냅니다. | **[Purdue 계층 그룹](#p)** |
| **디바이스 인벤토리 - 센서** | 디바이스 인벤토리는 Defender for IoT에서 검색한 광범위한 디바이스 특성을 표시합니다. 다음에 대한 옵션을 사용할 수 있습니다.<br /><br />- 표시된 정보 필터링<br /><br />- 이 정보를 CSV 파일로 내보내기<br /><br />- Windows 레지스트리 세부 정보 가져오기 | **[그룹](#g)** <br /><br />**[디바이스 인벤토리 - 온-프레미스 관리 콘솔](#d)** |
| **디바이스 인벤토리 - 온-프레미스 관리 콘솔** | 연결된 센서의 디바이스 정보는 디바이스 인벤토리에 있는 온-프레미스 관리 콘솔에서 볼 수 있습니다. 그러면 온-프레미스 관리 콘솔의 사용자에게 모든 네트워크 정보에 대한 포괄적인 보기가 제공됩니다. | **[디바이스 인벤토리 - 센서](#d)<br /><br />[디바이스 인벤토리 - 데이터 통합자](#d)** |
| **디바이스 인벤토리 - 데이터 통합자** | 온-프레미스 관리 콘솔의 데이터 통합 기능을 사용하면 다른 엔터프라이즈 리소스의 정보를 사용하여 디바이스 인벤토리의 데이터를 향상시킬 수 있습니다. 예제 리소스는 CMDB, DNS, 방화벽 및 Web API입니다. | **[디바이스 인벤토리 - 온-프레미스 관리 콘솔](#d)** |

## <a name="e"></a>E

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **엔진** | Defender for IoT의 셀프 학습 분석 엔진을 사용하면 서명을 업데이트하거나 규칙을 정의할 필요가 없습니다. 이 엔진은 ICS 특정 동작 분석과 데이터 과학을 사용하여 변칙, 맬웨어, 운영 문제, 프로토콜 위반 및 기준 네트워크 작업과의 차이가 있는지 네트워크 트래픽을 지속적으로 분석합니다.<br /><br />엔진에서 차이를 감지하면 경고가 트리거됩니다. **경고** 화면 또는 SIEM에서 경고를 보고 관리할 수 있습니다. | **[경고](#glossary-a)** |
| **엔터프라이즈 보기** | Defenders for IoT 센서가 설치된 비즈니스 단위, 사이트 및 영역을 나타내는 글로벌 맵입니다. 악의적인 경고, 운영 경고 등의 지리적 위치를 확인합니다. | **[사업부](#b)<br /><br />[사이트](#s)<br /><br />[영역](#z)** |
| **이벤트 타임라인** | 다음을 포함하여 네트워크에서 검색된 작업 타임라인<br /><br />- 경고가 트리거됨<br /><br />- 네트워크 이벤트(정보용)<br /><br />- 사용자 작업(예: 로그인, 사용자 삭제 및 사용자 생성) 및 경고 관리 작업(예: 음소거, 학습 및 승인) 센서 콘솔에서 사용할 수 있습니다. | - |
| **제외 규칙** | Defender for IoT에 기간, 디바이스 주소, 경고 이름 또는 특정 센서에 따라 경고 트리거를 무시하도록 지시합니다.<br /><br />예를 들어, 특정 센서에서 모니터링되는 모든 OT 디바이스가 오전 6시 30분부터 10시 15분 사이에 유지 관리 절차를 거치는 경우 이 센서가 미리 정의된 기간에 경고를 전송하지 않도록 하는 제외 규칙을 설정할 수 있습니다. | **[경고](#glossary-a)<br /><br />[경고 이벤트 음소거](#m)** |

## <a name="f"></a>F

| 용어 | Description | 자세한 정보 |

## <a name="g"></a>G

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **그룹** | 프로그래밍 작업을 수행한 디바이스 및 특정 서브넷에 있는 디바이스와 같이 특정 특성을 포함하는 미리 정의된 디바이스 또는 사용자 지정 디바이스 그룹입니다. 그룹을 사용하면 디바이스를 보고 Defender for IoT에서 분석하는 데 유용합니다.<br /><br />그룹은 디바이스 맵과 디바이스 인벤토리를 통해 보고 만들 수 있습니다. | **[디바이스 맵](#d)<br /><br />[디바이스 인벤토리](#d)** |

## <a name="h"></a>H

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **Horizon Open Development Environment** | 독점 및 사용자 지정 프로토콜이나 표준에서 벗어난 프로토콜을 실행하는 IoT 및 ICS 디바이스를 보호합니다. Horizon ODE(Open Development Environment) SDK를 사용하여 정의된 프로토콜을 기반으로 네트워크 트래픽을 디코딩하는 디섹터 플러그 인을 개발할 수 있습니다. Defenders for IoT 서비스는 트래픽을 분석하여 완전한 모니터링, 경고 및 보고 기능을 제공합니다.<br /><br />다음 상황에서 Horizon을 사용합니다.<br /><br />- 플랫폼 버전을 업그레이드할 필요 없이 가시성 및 제어를 **확장** 합니다.<br /><br />- 외부 플러그 인으로 사이트를 개발하여 독점 정보를 **보호** 합니다.<br /><br />- 경고, 이벤트 및 프로토콜 매개 변수에 대한 텍스트를 **지역화** 합니다.<br /><br />자세한 내용은 고객 성공 담당자에게 문의하세요. | **[프로토콜 지원](#p)<br /><br />[지역화](#l)** |
| **Horizon 사용자 지정 경고** | Horizon Framework 트래픽 디섹터를 기준으로 하는 모든 프로토콜에 대한 사용자 지정 경고를 트리거하여 기업의 경고 관리 수준을 높입니다.<br /><br />다음 경고는 정보를 전달하는 데 사용할 수 있습니다.<br /><br />- 독점 Horizon 플러그인에서 프로토콜 및 기본 프로토콜을 기준으로 하는 트래픽 감지 정보<br /><br />- 모든 프로토콜 계층의 프로토콜 필드 조합 정보 | **[프로토콜 지원](#p)** |

## <a name="i"></a>I

| 용어 | Description | 자세한 정보 |
|--|--|--|
| **통합** | 파트너 시스템과 디바이스 정보를 공유하여 Defender for IoT 기능을 확장합니다. 조직은 이전에 격리된 보안, NAC, 사고 관리 및 장치 관리 솔루션을 연결하여 시스템 전반의 대응을 가속화하고 위험을 보다 신속하게 완화할 수 있습니다. | **[전달 규칙](#f)** |
| **내부 서브넷** | Defender for IoT에서 정의한 서브넷 구성입니다. 퍼블릭 범위를 내부 범위로 사용하는 환경과 같은 일부 경우에는 Defender for IoT에 모든 서브넷을 내부 서브넷으로 확인하도록 지시할 수 있습니다. 서브넷은 맵 및 다양한 Defender for IoT 보고서에 표시됩니다. | **[서브넷](#s)** |
