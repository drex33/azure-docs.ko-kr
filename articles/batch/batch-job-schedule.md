# <a name="schedule-jobs-for-efficiency"></a>효율성을 위해 작업 예약

Batch 작업을 예약하면 [다른 태스크에 대한 종속성이 있는 태스크](batch-task-dependencies.md)를 고려하면서 먼저 실행하려는 작업의 우선 순위를 지정할 수 있습니다. 작업을 예약하면 최소한의 리소스를 사용하고 있는지 확인할 수 있습니다. 필요하지 않은 경우 노드를 해제할 수 있으며 다른 태스크에 종속된 태스크는 워크플로를 최적화하는 동안 적당한 시기에 분리됩니다. 한 번에 하나의 작업만 실행되고 이전 작업이 완료될 때까지 새 작업이 시작되지 않으므로 작업을 자동 완성으로 설정할 수도 있습니다.

작업 관리자 태스크를 사용하여 예약하는 태스크는 작업과 연결됩니다. 작업 관리자 태스크는 작업의 태스크를 만듭니다. 이렇게 하려면 작업 관리자 태스크가 Batch 계정으로 인증해야 합니다. AZ_BATCH_AUTHENTICATION_TOKEN 액세스 토큰을 사용하세요. 이 토큰은 작업의 나머지 부분에 대한 액세스를 허용합니다.

Azure CLI를 사용하여 작업을 관리하려면 [az batch job-schedule](/cli/azure/batch/job-schedule)을 참조하세요. Azure Portal에서 작업 일정을 만들 수도 있습니다.

## <a name="schedule-a-job-in-the-azure-portal"></a>Azure Portal에서 작업 예약

1. [Azure Portal](https://portal.azure.com/)에 로그인합니다.
1. 작업을 예약하는 데 사용할 Batch 계정을 선택합니다.
1. 왼쪽 탐색 창에서 **작업 일정** 을 선택합니다.
1. 새 작업 일정을 만들려면 **추가** 를 선택합니다.
1. **기본 양식** 에서 다음 정보를 입력합니다.
   - **작업 일정 ID**: 이 작업 일정의 고유 식별자입니다.
   - **표시 이름**: 이 이름은 선택사항이며 고유할 필요는 없습니다. 최대 길이는 1024자입니다.
   - **다음 시간까지 실행하지 않음**: 작업이 실행될 가장 빠른 시간을 지정합니다. 이를 설정하지 않으면 작업이 즉시 실행될 수 있습니다.
   - **다음 이후 실행 안 함**: 여기에 입력한 시간 이후에는 작업이 실행되지 않습니다. 시간을 지정하지 않으면 명시적으로 종료할 때까지 활성 상태로 유지되는 되풀이 작업 일정이 생성됩니다.
   - **반복 간격**: 작업 사이의 시간을 지정하려면 **사용** 을 선택합니다. 예약된 시간에 하나의 작업만 실행할 수 있으므로 작업 일정 아래에 새 작업을 만들어야 하는 시점에 이전 작업이 아직 실행되고 있으면 Batch 서비스는 이전 작업이 완료될 때까지 새 작업을 만들지 않습니다.
   - **시작 창**: 작업을 만들어야 하는 시간 간격을 지정하려면 **사용자 지정** 을 선택합니다. 이 창에서 작업이 만들어지지 않으면 다음 일정이 반복될 때까지 새 작업이 만들어지지 않습니다.

     :::image type="content" source="media/batch-job-schedule/add-job-schedule-02.png" alt-text="Azure Portal에서 작업 일정 추가 옵션의 스크린샷.":::  

1. 기본 양식 하단에서 작업을 실행하려는 풀을 지정합니다. Batch 계정의 풀 목록에서 선택하려면 **업데이트** 를 선택합니다.
1. **풀 ID** 와 함께 다음 정보를 입력합니다.
   - **작업 구성 태스크**: **업데이트** 를 선택하여 작업 관리자 태스크와 작업 준비 태스크 및 작업 릴리스 태스크(사용 중인 경우)의 이름을 지정하고 구성합니다.
   - **표시 이름**: 이 이름은 선택사항이며 고유할 필요는 없습니다. 최대 길이는 1024자입니다.
   - **우선 순위**: 슬라이더를 사용하여 작업의 우선 순위를 설정하거나 상자에 값을 입력합니다.
   - **최대 벽시계 시간**: 작업을 실행할 최대 시간을 설정하려면 **사용자 지정** 을 선택합니다. 그렇게 하면 해당 시간 프레임 내에 완료되지 않는 경우 Batch에서 작업을 종료합니다.
   - **최대 태스크 다시 시도 횟수**: 태스크를 다시 시도할 수 있는 횟수를 지정하려면 **사용자 지정** 을 선택하고 필요한 만큼 태스크를 시도하려면 **무제한** 을 선택합니다. 이는 API 호출을 재시도할 수 있는 횟수와 동일하지 않습니다.
   - **모든 태스크가 완료될 때**: 기본값은 NoAction이지만 모든 태스크가 완료되었을 때 작업을 종료하려는 경우(또는 작업에 태스크가 없는 경우) **TerminateJob** 을 선택할 수 있습니다.
   - **태스크가 실패한 경우**: 재시도 횟수가 소진되었거나 태스크를 시작할 때 오류가 발생한 경우 태스크가 실패합니다. 기본값은 NoAction이지만 실패할 경우 태스크의 종료 조건과 관련된 작업을 수행하려는 경우 **PerformExitOptionsJobAction** 을 선택할 수 있습니다.

     :::image type="content" source="media/batch-job-schedule/add-job-schedule-03.png" alt-text="Azure Portal의 새 작업 일정에 대한 작업 사양 옵션의 스크린샷.":::

1. **저장** 을 선택하여 작업 일정을 만듭니다.

작업 실행을 추적하려면 **작업 일정** 으로 돌아가서 작업 일정을 선택합니다. 세부 정보를 보려면 **실행 정보** 를 확장합니다. 이 화면에서 작업 일정을 종료, 삭제 또는 사용하지 않도록 설정할 수도 있습니다.
